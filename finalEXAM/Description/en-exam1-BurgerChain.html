Exercise

In this exercise, we represent a restaurant chain that sells burgers.

Order

During instantiation, all orders get a unique orderNumber starting at 1. Use nextOrderNumber to keep track of it.

The price is calculated in the constructor, too.

  • The base cost of each hamburger is 2490 Ft
  • Asking it in a menu adds 590 Ft to it.
    • The total cost this way is 3080 Ft.
  • The additional cost of a DELIVERY is 990 Ft.
    • A burger delivered costs 3480 Ft without a menu, 4070 Ft with it.

The isDone field and its getter show whether the order is ready. It's false initially.

You can get the order ready by calling make(). If it is already ready, let the method throw an IllegalStateException.

Testing

Let restaurant.test.OrderTest.testOrder() use a parameterized JUnit 5 tester to check for burgers of all four possible prices (in a menu? with delivery?) that the price is computed properly.

Let restaurant.test.OrderTest.testMake() test an order in the following way.

  • Test that it isn't immediately done.
  • ... but after calling make(), it is.
  • Calling make() again throws the expected exception.
    • Surround this call with exception handling. Leave the code of the catch clause empty.
    • After making the call, immediately call fail() (a built-in of JUnit 5). This shows an error in case the exception is not thrown.

Restaurant and BurgerPlace: a simple restaurant

Use a default constructor in BurgerPlace.

Let orders map order numbers to ordered burgers.

Let takeOrder() do the following.

  • It checks the following condition and throws if needed.
    • IllegalArgumentException if orderNumber is not known in orders.
    • IllegalArgumentException if DELIVERY was asked for.
    • IllegalStateException if the order is not done yet.
  • Otherwise, do the following.
    • Remove the given order.
    • Increase profit by the price.
    • Return the ordered item.

The restaurant prepares the first order that is not done yet using makeNextOrder(). The method returns if a new burger was prepared.

Testing

Test ordering using restaurant.test.BurgerPlaceTest.testOrderProcess(). Take the following steps.

  1. Create a new BurgerPlace.
    • Check that its profit is 0 Ft.
  2. Order a "Bacon Burger" as a TAKEAWAY, no menu.
    • Check that it is not done yet.
  3. Also order a "Double Cheeseburger" with menu as DINE_IN.
    • Check that the order number is one more than that of the previous order.
    • Check that it is not done yet.
  4. Prepare the first order.
    • Check that the first order is done but the second one isn't.
    • Check that the profit is still at 0 Ft.
  5. Take the first order.
    • Check that the profit is now at 2490 Ft.
  6. Prepare the second order.
    • Check that it is done.
  7. Take it.
    • Check that the profit is now at 5570 Ft.
  8. Try to prepare one more order.
    • This has to return false.

BurgerChain

This sort of a restaurant has its own name and keeps tabs on delivered orders.

Taking an order is done as previously, but before returning, add the ordered burger into pastOrders.

Two burger chains are considered equal if their names and the number of their pastOrders match.

The textual representation of a burger chain looks like this. This example shows orders similar to those made in the previous section's test. Use '\n' for line ends.

Best Burgers
#1. Bacon Burger - without a menu (TAKEAWAY)
#2. Double Cheeseburger - in a menu (DINE_IN)

Use saveData() to save the burger chain's textual representation into a file.

  • Call the file data-{chain's name}.txt.
    • Example: data-Best Burgers.txt

Testing

Test the above using restaurant.test.BurgerChainTest.testOrderProcess().

  • Create two burgers in the burger chain Best Burgers like before and let people buy them.

  • Test the textual representation.

    • Hint: here is a simple way to create a multiline text. In it, line ends are \n characters.
        String expected = """
Best Burgers
#1. Bacon Burger - without a menu (TAKEAWAY)
#2. Double Cheeseburger - in a menu (DINE_IN)
    """;

  • Let the tester also save the chain into a text file.

    • You can manually inspect its contents, no automated testing is required here.

In restaurant.test.BurgerChainTest.testTwoPlaces(), do the following.

  • Create two burger chains with the same name.
    • Check that they're equal.
  • Add the previous "Bacon Burger" order to the first, and the "Double Cheeseburger" to the other. Then make the orders in both. Then take the orders in both.
    • For all steps, they chains have to be equal to each other.
  • If the order numbers give you trouble, do the following.
    • Place this annotation on the tester class: @TestMethodOrder(org.junit.jupiter.api.MethodOrderer.OrderAnnotation.class)
    • Place @Order(1) and @Order(2) on the respective tester methods.
